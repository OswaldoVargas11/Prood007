# Catálogo de herramientas del agente de IA — roadmap

Generado por workflow de descubrimiento (188 capacidades mapeadas → 89 tools propuestas). Excluye, por ICP: fiscal/billing (escritura), litigios/LexNET, platform/super-admin, borrados destructivos. Las 11 tools actuales no se repiten.

## Flujos (procesos multi-paso que el agente orquesta)

- **Alta de cliente y apertura de expediente (con control de conflicto)** _(requiere confirmación)_ — _trigger:_ «Da de alta a [cliente] y abre un expediente de [asunto]»
  - `check_conflict_of_interest -> (si limpio) create_client -> create_matter -> apply_presentation_to_matter (checklist del tipo de tramite) -> save_engagement_letter -> opcional generate_document_package (set de intake)`
- **Conversion de lead a cliente activo** _(requiere confirmación)_ — _trigger:_ «Convierte el lead de [nombre] en cliente y abrele expediente»
  - `get_lead_detail -> check_conflict_of_interest -> convert_lead_to_client (createMatter=true) -> upsert_client_kyc (estado inicial de diligencia) -> create_client_portal_user (invitacion al portal)`
- **Montaje de operacion transaccional (deal kickoff)** _(requiere confirmación)_ — _trigger:_ «Prepara la operacion de M&A del expediente [ref]: partes, hitos y data room»
  - `get_matter -> add_transaction_party (varias: buy-side/sell-side) -> add_transaction_milestone (SIGNING/CLOSING/LONGSTOP) -> create_data_room -> add_data_room_folder (estructura DD) -> add_data_room_group (vendedor/comprador) -> create_data_room_grant (enlaces a las partes)`
- **Preparacion y cierre de operacion (closing)** _(requiere confirmación)_ — _trigger:_ «Prepara el cierre del expediente [ref] y genera el binder»
  - `create_closing_checklist (plantilla M&A) -> get_closing_checklist_detail -> update_closing_item (marcar condiciones satisfechas/eximidas) -> get_disclosure_schedules + add_disclosure_schedule (cerrar reps & warranties) -> get_registry_filings (verificar tramites) -> generate_closing_binder`
- **Plazo procesal desde notificacion (calculo y encargo)** _(requiere confirmación)_ — _trigger:_ «Me ha llegado una notificacion en [expediente], calcula el plazo y crea la tarea»
  - `get_matter -> preview_task_from_deadline (dias habiles + festivos del despacho) -> (mostrar fecha al letrado) -> create_procedural_task -> reassign_task (si procede balancear carga)`
- **Revision semanal de cartera y cobranza operativa (no fiscal)** _(requiere confirmación)_ — _trigger:_ «Como va el despacho, que tengo estancado o pendiente esta semana»
  - `firm_overview -> list_stale_matters_report -> list_open_tasks -> list_matter_checklists (completitud de tramites en curso) -> create_saved_view (guardar el filtro de seguimiento)`
- **Redaccion asistida de documentacion de expediente** _(requiere confirmación)_ — _trigger:_ «Redacta el paquete de documentos / contrato para el expediente [ref]»
  - `get_matter -> list_document_packages o list_templates -> get_template_detail (validar tokens) -> list_clauses (insertar fragmentos pre-aprobados) -> generate_document_package -> create_document_folder + move_document_to_folder (organizar borradores)`

## Tarjetas (Generative UI) sugeridas

- `check_conflict_of_interest`: Tarjeta de semaforo: verde 'Sin conflicto' o rojo 'Conflicto detectado' con lista de coincidencias (cliente/contraria + referencia de expediente enlazable).
